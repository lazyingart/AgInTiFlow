# AgInTiFlow Product Positioning

## Recommended Headline

Low-cost, project-aware agents for real-life problems.

## Recommended Subtext

Use the same agent workspace from Web or CLI, with DeepSeek/Venice/OpenAI routing, visible tool calls, durable sessions, scouts, SCS supervision, AAPS workflows, and guarded local execution.

## Why This Is Better Than "Web-First"

`web-first` describes an implementation preference, not the user value. It can also make the CLI sound secondary even though AgInTiFlow is strongest when the same project session is available from both terminal and browser.

The new line leads with three product-level promises:

- Low-cost: model routing, DeepSeek Flash/Pro, and scouts make agent work economically practical.
- Project-aware: the working folder, files, sessions, artifacts, and tools are first-class state.
- Real-life problems: coding, writing, review, research, documents, images, Android/app tasks, maintenance, and longer workflows are the intended target, not only chat.

## Messaging Hierarchy

1. Tagline: explain the category and value in one sentence.
2. Subtext: name the concrete differentiators: Web + CLI, model routing, visible tools, durable sessions, scouts, SCS, AAPS, guarded execution.
3. Proof points: show screenshots, CLI commands, session resume, Docker policy, SkillMesh, AAPS adapter, image generation, and model selectors.
4. Technical depth: keep detailed command lists, storage layout, and philosophy notes lower in the README and docs.

## Terms To Prefer

- Use `Web + CLI`, `same workspace`, or `shared project session` instead of `web-first`.
- Use `project-aware` instead of generic `AI workspace`.
- Use `visible tool calls` or `inspectable execution` instead of vague `transparent`.
- Use `guarded local execution` instead of only `safe shell`, because it covers files, shell, Docker, installs, secrets, and session logs.
- Use `low-cost routing` instead of only naming models, then cite DeepSeek/Venice/OpenAI as proof.

## Product Pillars

| Pillar | User-facing meaning |
| --- | --- |
| Low-cost routing | DeepSeek/Venice/OpenAI roles let routine work stay cheap and complex work escalate only when needed. |
| Project-aware workspace | The agent works inside a real folder with durable sessions, artifacts, file diffs, and resumable history. |
| Inspectable tools | Plans, tool calls, shell output, file changes, and artifacts are visible instead of hidden behind chat text. |
| Scouts | Parallel scouts cheaply map architecture, risks, tests, and context before expensive execution. |
| SCS supervision | Student-Committee-Supervisor mode adds a typed planning and evidence gate for complicated tasks. |
| AAPS workflows | Large work can be described top-down as agentic pipeline scripts and executed through AgInTiFlow. |
| Guarded execution | Docker, path guardrails, install policy, secret redaction, and blocked destructive commands keep local work practical. |
